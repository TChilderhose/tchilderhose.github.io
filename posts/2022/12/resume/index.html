Resume Revamp

2022-12-02   5-minute read   resume • hugo

We’ve currently been hiring at my work and I’ve seen about 200+ resumes in a short amount of time. While reviewing them, I made the realization that the visuals of resume effected me more than I thought it would. Which made me also realize it might be time to update and upgrade my resume both in terms of visuals and flow.

Current Resume Flow and Status

I essentially have 3 resumes that I have been keeping up to date.

  1. The “real” resume that I would submit to places which is in Google Documents that I export as pdf.
  2. My Linkedin experience.
  3. The resume page on this site.

This took a bit of effort to all keep updated and in sync which was partially the reason it’s fallen to the wayside.

Ideally, I would like a single place I can update and use; but I wont be able to automate the LinkedIn resume. But I would like to some how combine the resume on this site and my “real” resume, with the idea of being to just print my resume page to pdf and having it look good.

Current Resume Page

This is what the resume on my site looked like. Overall, not great. Immediately hit with a bullet point list with no real good organization and wasted space.

Lets see what it looks like when we try to print.

So having it “just” work wont be an option unfortunetly and will need a whole revamp.

Research

While I like the general theme of my site, its not really a good theme for a resume. I would really like it to appear like a paper resume just on the web.

The one great thing about using Hugo as a website frawework, it has a lot of community driven themes, even resume themes. The one theme that caught my eye as a good starting point was a theme called Resume A4.

It wasn’t exactly what I wanted, but like I said was a very good starting point.

Hugo Resume Theme Intial Setup

The one catch of using Hugo is that it doesnt really support multiple themes for a site. So to get around this I am compiling essentailly 2 sites with their own configs and packaging them together.

./bin/hugo --config=config.yaml --minify
./bin/hugo --config=config_resume.yaml --minify

While ensuring their baseURL are set correctly.

config.yaml

baseURL: '/'

config_resume.yaml

baseURL: "https://tchilderhose.ca/resume/"

This now allows me to have my main blog website theme apply everywhere except for /resume.

Data and Styling

Now that I had the theme running along side my main site theme, the next thing was to work on organizing some of the data into files that I could easily manage. I settled on 4 yaml files that I will use to organize my data that will populate my resume.

features.yaml - Mostly general non-professional information about me.

experience.yaml - All of my work experience in a very easy and flexible way to maintain.

education.yaml - Education information.

projects.yaml - Any non-professional projects that I work on.

Very happy with how the structure turned out.

Next was styling, which was really just a lot of playing around with css and adding different elements to personal taste. I wont go over everything I adjusted but I really enjoy the “Tag Box” of the tech I used for certain projects. One of the resumes I reviewed at my work had this for their general skills and I was really drawn to it.

After all my tweaking I was very happy how my new site resume was looking.

Printing

Now to focus on the main reason of this switch, printing.

Luckily there is a css tag to style specific things for print and really not a lot was needed to get it looking good.

  1. Need to set the page size.
  2. Need to hide the header and footer information from being printed. (current date, page name, page numbet, etc)
  3. Make sure different elements are not being clipped on one page and then printed on the next.

This is the entirety of the css file for print css.

@media print {  
  @page {
    width: 8.5in;
    height: 11in;
    margin: 0.3in 0;
  }

  @page :first {
    margin-top: 0;
  }

  #paper {
    -moz-box-shadow: none;
    -webkit-box-shadow: none;
    box-shadow: none;
    margin: 0;
    padding: 0;
  }

  .item {
    break-inside: avoid;
    page-break-inside: avoid;
  }

  footer {
    display: none;
  }
}

There is still a few minor tweaks I would like to make when I have more time, but I think for now it does a really good job.

New Resume Flow

So now my new flow for my resume is to:

  1. Update the yaml files.
  2. Deploy the site.
  3. Copy and paste sections from the new resume page into LinkedIn.
  4. When needed, open the new resume page in my browser and save it as a pdf.

Managing my resume in a more technical way in yaml files has really pleased the techy in me. I know its not for everyone, but it makes a lot of sense in my head and I’m very happy with the new style and streamlined flow of my resume.
